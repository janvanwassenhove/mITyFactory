## Domain Concepts

### User
A person who interacts with the application.

### Account
A user's identity and associated data within the system.

### Session
An authenticated period of user activity.

### Workspace
A logical grouping of user resources.

---

## Technical Terms

### API
Application Programming Interface. The contract for interacting with the system.

### Endpoint
A specific URL path that handles requests.

### Request
An incoming message from a client.

### Response
An outgoing message to a client.

---

## Data Terms

### Entity
A domain object with a unique identity.

### Repository
An abstraction for data persistence operations.

### Migration
A versioned change to the data schema.

### Seed
Initial data loaded into the system.

---

## Process Terms

### Pipeline
A series of processing stages.

### Job
A unit of background work.

### Queue
A buffer for jobs awaiting processing.

### Worker
A process that executes jobs from a queue.
